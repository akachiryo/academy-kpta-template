# KPTA ボード

KPTA ボードは、チームのレトロスペクティブを効率的に実施するための GitHub プロジェクトボードです。

## ボードの構成

- **Thanks**: チームや個人に対する感謝を表明するカラム
- **Keep**: 継続したい良い習慣や取り組み
- **Problem**: 解決すべき課題や問題点
- **Try**: 試してみたい新しいアイデアや改善案
- **妨害リスト**: プロジェクトの進行を妨げている要因
- **Done**: 完了したタスクや解決した問題

## レトロ(KPT)進めかた

1. **レトロスペクティブの終了時間の確認**
   - 目安は`30分`
2. **`Try`：チケットの確認【5 分】**
   - 解決済みのチケットは`Done`レーンに移動する
   - 継続させたいチケットは`Keep`レーンに移動する
3. **`Keep`：チケットの追加【3 分】**
   - 開発メンバー全員で、より良くする余地がある場合は`Keep`レーンに追加する
4. **`Keep`：チケットの確認【10 ~ 15 分】**
   - 既に定着・浸透しているチケットは`Done`レーンに移動する
   - 継続させたいチケットは`Keep`レーンに移動する
5. **`Problem`：チケットの追加【3 分】**
   - 開発メンバー全員で、前スプリントで問題だった事項について`Problem`レーンに追加する
6. **`Problem`：チケットの優先順位を決める【1 分】**
   - 各開発メンバーが問題に思うチケットにアイコンをつける
7. **`Problem`：優先度順にチケットの検討【25 ~ 30 分】**
   - 開発メンバー全員で優先度が高い順に検討し、解決策を`Try`レーンに追加する
   - `Try`レーンに追加するチケットには、「#{Problem のチケットの番号}」の形で備考に記載をし紐づけを行うこと
   - すべてのチケットについて検討する必要はなく、時間がなくなった場合は検討を行わない
8. **`Problem`：チケットの削除**
   - `Problem`レーンにあるチケットをすべて削除する
9. **【任意】`妨害リスト`：チケットの検討**
   - 時間が余った場合に行う
   - 開発メンバー全員でチケットについて検討する
